Fully Disconnected (air-gapped)
If you plan to install OpenShift in a fully disconnected (air-gapped) environment, Aba can download all required components—including
the Quay mirror registry install file, container images, and CLI install files—and package them into an install bundle that you can
transfer into your disconnected environment.
[ABA] Install OpenShift into a fully disconnected network environment? (Y/n): n

Partially Disconnected
A mirror registry can be synchronized directly from the Internet, allowing OpenShift to be installed from the mirrored content.
[ABA] Install OpenShift from a mirror registry that is synchonized directly from the Internet? (Y/n): Y

Instructions for synchronizing images directly from the Internet to a mirror registry
Set up the mirror registry and sync it with the necessary container images.
To store container images, Aba can install the Quay mirror appliance or you can use an existing container registry.

Run:
  aba -d mirror install                # to configure an existing registry or install Quay.
  aba -d mirror sync --retry <count>   # to synchronize all container images - from the Internet - into your registry.

Or run:
  aba -d mirror sync --retry <count>   # to complete both actions and ensure any image synchronization issues are retried.

  aba mirror --help                    # See for help.
After the images are stored in your mirror registry, proceed with the OpenShift installation.
